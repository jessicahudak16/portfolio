Mirrored Robots Demo

Overview

In 2022, the release of ForgeOS 5.2.0 introduced a new app: Rule Engine. This app opened the door for running logic in the background of any programmed tasks. I did extensive testing on this new feature, both to test for bugs and to gain a thorough understanding of how the app works so that I could accurately update the ForgeOS user manual. While pushing Rule Engine to its limits, I set up the unconventional scenario of having one robot “mirror” a hand-guided robot.

How it Works

Two robots (a small UR3e and a large UR10e) are hooked up on the same instance of ForgeOS. Both robots use a similar coordinate system (e.g., joint coordinates of 0,0,0,0,0,0 dictates a similar configuration).

Steps:

1. In the Rule Engine app, create a linked values rule where:

UR10e’s waypoint “Ur3e position” = UR3e’s parameter “Current Robot Position”

2. In a task, check if:

UR10e's waypoint “UR3e position” = UR10e’s parameter “Current Robot Position”
  • If TRUE, loop back to check again.
  • If FALSE, the UR10e does a joint move to “UR3e position”, then loop back to check again.

3. Put the UR3e in HAND GUIDE mode.

4. Put the UR10e in RUN mode.

5. Start the rule and task.
